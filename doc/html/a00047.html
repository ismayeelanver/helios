Detailed Description

This class implements a window manager.

It is responsible for managing the windows on the screen and handling events such as key presses.

void WindowManager::run ( )

Starts the window manager.

The main loop of the window manager.

This function is called when the window manager is started. It sets up the event loop, which is the main loop of the window manager. This loop will continue to run until the window manager is stopped.

Waits for events and handles them accordingly. Events are handled in the following order:

  1. MapRequest - Maps the window and adds it to the list of managed windows.
  2. UnmapNotify - Unmaps the window and removes it from the list of managed windows.
  3. DestroyNotify - Destroys the window and removes it from the list of managed windows.
  4. EnterNotify - Sets the input focus to the window that the pointer is currently over.
  5. KeyPress - Switches to the specified workspace when a number key with the Mod4 modifier is pressed.

If the event is not one of the above, the window manager will log an error message and break out of the main loop.

The window manager will also log an error message if the event is invalid.

void WindowManager::tile_windows ( )
private

Tiles all windows in the current workspace.

Tile all windows in the list in a non-overlapping manner across the screen.

The tiling algorithm is as follows:

  1. Split the screen in half horizontally.
  2. Place the first window in the top half of the screen.
  3. For each subsequent window: a. Find the window with the largest area. b. Split that window in half, either horizontally or vertically, depending on the previous split direction. c. Place the new window in the newly created space. d. Switch the split direction.
